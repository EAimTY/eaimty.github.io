博客搬到了 VPS 上，顺便更新一下主题
2016-04-23

总结一下对服务器安全的简单设置。

最近，~~墙又高了很多，中国电信开始大量墙境外主机和 VPS 的 IP，~~OpenShift 在国内访问很不稳定，经常出现 timeout，再加上 OpenShift 用的是 Apache，本身就不擅长处理动态程序，导致博客频频 503...实在是不能忍了...突然想起前些天手贱多买了一台 VPS，于是折腾了一下，把博客搬到了搬瓦工的洛杉矶机房。 这回从国内访问的速度快多了，不过貌似 CloudFlare 的 DNS 已经快要被墙了...🤣

然而在向新服务器导入 MySQL 数据库的时候忘记先删除安装时自动新建的数据库了，也就是把 utf8mb4 编码的数据库导入成了 uft8... 结果就是排版和 emoji 都没了...没办法，重新排吧┐(‘д’)┌

转移过程中顺便改了改主题，添加了一些功能，改了一下配色，顺便更新了一下 bootstrap 和 jquery（没办法，强迫症┐(‘д’)┌），压缩了一下 Material-ScrollTop 的 css 和 js，还把几乎所有代码的格式统一了，显得很规整（没办法，强迫症┐(‘д’)┌）
